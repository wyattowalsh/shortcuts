### HTML Runtime

- `local-bundled`: `constrained`. The shortcut resolves every input type to local render HTML text before opening Safari. Single files are optimized for normal UTF-8 HTML. ZIP and folder inputs also attempt best-effort same-name asset rewriting. Use `shortcutkit html bundle` for deterministic inlining of CSS, JavaScript, images, fonts, manifests, CSS url() dependencies, and srcset into a single self-contained HTML file.
- `hosted`: `supported` via `here-now`. The shortcut always shows the here.now deployment command after the local render attempt. Use `shortcutkit html publish --provider here-now --dry-run` to plan a hosted multi-file Safari runtime; `--run` returns a shareable siteUrl and requires explicit approval.

#### Local Bundle

- Strategy: `resolved-html-text-plus-best-effort-same-name-data-url-rewrite`
- Rendered scheme: `data:text/html;base64`
- Deterministic command: `uv run shortcutkit html bundle path/to/index.html --output path/to/index.bundled.html --json`
- Supported asset extensions: `css`, `js`, `mjs`, `png`, `jpg`, `jpeg`, `gif`, `webp`, `avif`, `svg`, `json`, `webmanifest`, `woff`, `woff2`, `ttf`, `otf`
- Limitations:
  - Single-file rendering resolves HTML as text and is optimized for normal UTF-8 HTML files, not arbitrary legacy encodings.
  - Same-name replacement is best-effort and does not understand every nested path form.
  - Use shortcutkit html bundle for CSS url(), @import, srcset, deterministic hashing, and large-site diagnostics.
  - data: URL rendering can hit iOS or Safari size limits.
