# Getting started

1. Create a shortcut package with `shortcut.yml`.
2. Declare source mode, compatibility, permissions, tests, and review metadata.
3. Run `shortcutkit validate`.
4. Run `shortcutkit security audit --strict`.
5. Generate catalog and docs references.

The first release should prefer correctness and explicit review notes over broad automation claims.
