# Capability: `app-intents-lab`

## Purpose

Provide Swift examples and docs for App Intents/App Shortcuts integration.

## Requirements

### Requirement: Swift Examples

The lab MUST include minimal and parameterized AppIntent examples.

#### Scenario: swift-examples

- GIVEN a developer opens snippets WHEN reading docs THEN they can copy a minimal intent.

### Requirement: AppShortcutsProvider

The lab MUST include AppShortcutsProvider and phrase-design examples.

#### Scenario: appshortcutsprovider

- GIVEN a demo app builds WHEN shortcuts are discovered THEN phrases are documented.

### Requirement: Entities

The lab MUST include typed App Intents examples and MUST document AppEntity, EntityQuery, and AppEnum patterns or their current limitations.

#### Scenario: entities

- GIVEN an entity-backed intent is shown WHEN docs render THEN query behavior is explained.

### Requirement: Testing Guidance

The lab MUST document testing and availability constraints.

#### Scenario: testing-guidance

- GIVEN a developer follows docs WHEN Xcode version differs THEN availability notes guide adaptation.
